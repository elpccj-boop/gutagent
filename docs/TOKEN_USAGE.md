# Token Usage Reporting - Cross-Provider Comparison

All providers now use **normalized semantics** matching Claude's API response format.

## Unified Token Fields

| Field | Description |
|-------|-------------|
| `input_tokens` | Tokens NOT from cache (dynamic context, user message, conversation history) |
| `output_tokens` | Tokens generated by the model |
| `cache_creation_input_tokens` | Tokens written to cache (first request only) |
| `cache_read_input_tokens` | Tokens read from cache (subsequent requests) |

**Total input = `input_tokens` + `cache_creation_input_tokens` (or `cache_read_input_tokens`)**

## Provider Comparison

### Claude (Anthropic)

Claude's API already returns these exact fields - no normalization needed.

```
First request:   input_tokens=500, cache_creation_input_tokens=5000, output_tokens=200
Subsequent:      input_tokens=500, cache_read_input_tokens=5000, output_tokens=200
```

- Cache TTL: 5 minutes (ephemeral), 1 hour (with ttl="1h")
- Cache discount: 90% on cache reads, 25% extra on cache writes
- Minimum tokens: 1024

### Gemini (Google)

Gemini's API returns total input (`prompt_token_count`). We normalize:
- `input_tokens = prompt_token_count - cached_content_token_count`

```
First request:   input_tokens=789, cache_creation_input_tokens=4890, output_tokens=199
Subsequent:      input_tokens=997, cache_read_input_tokens=4890, output_tokens=551
```

- Cache TTL: 1 hour minimum
- Cache discount: 90% (Gemini 2.5+), 75% (Gemini 2.0)
- Minimum tokens: 2048 (Gemini 2.5), 4096 (Gemini 3)
- Explicit caching via `client.caches.create()`

### OpenAI

OpenAI's API returns total input (`prompt_tokens`). We normalize:
- `input_tokens = prompt_tokens - cached_tokens`

```
First request:   input_tokens=5000, output_tokens=200  (no cache_creation)
Subsequent:      input_tokens=500, cache_read_input_tokens=4500, output_tokens=200
```

- Cache TTL: 5-10 minutes (up to 1 hour off-peak)
- Cache discount: 50%
- Minimum tokens: 1024
- Automatic caching - no explicit API calls needed
- **No `cache_creation_input_tokens`** - OpenAI doesn't charge extra for cache writes

## What Each Field Represents in GutAgent

| Field | Content |
|-------|---------|
| `cache_creation_input_tokens` / `cache_read_input_tokens` | Static prompt (~6k tokens): system instructions, profile, tool definitions |
| `input_tokens` | Dynamic context + user message: recent meals, vitals, alerts, last exchange, conversation history |
| `output_tokens` | LLM response: text, tool calls |
